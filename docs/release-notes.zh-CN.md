# MiPadLink 发布说明草稿

## 概述

这个版本主要围绕三个方向推进：

- 让项目更适合公开展示
- 让启动流程更简单
- 让不同屏幕比例的安卓平板显示更合理

## 本次亮点

- 仓库文档整理成更适合公开展示的中英双版本
- 一键启动脚本更完整
- dashboard 支持更清晰的显示模式和采集档位控制
- 默认抓屏路径切到 macOS `screencapture`
- 安卓端新增 `完整显示 / 铺满裁切`
- 进一步收口虚拟显示器生命周期和触控/显示映射

## 用户可见变化

- `./start.sh` 现在会处理 `adb reverse`、host 启动和 dashboard 打开
- 安卓客户端新增：
  - `完整显示`
  - `铺满裁切`
  - 全屏进入
- dashboard 地址仍是 `http://127.0.0.1:9010`

## 对外表述建议

当前更适合这样介绍 MiPadLink：

- 一个开源的 USB 有线扩展屏原型
- 已在小米 Pad 7 上验证
- 架构上可扩展到很多安卓平板

当前不建议这样表述：

- 所有安卓平板都天然即插即用
- 已经达到所有品牌都可稳定商用的程度

## 已知限制

- 目前还是 JPEG 推流，后面还有延迟和 CPU 优化空间
- 不同安卓厂商的系统策略仍可能影响稳定性
- 要覆盖更多平板，还需要继续做兼容档位

## 下一版值得推进的方向

- 硬件编码视频链路
- USB 重连后的稳态恢复
- 更完整的设备兼容性矩阵
- 预编译 APK 发布
